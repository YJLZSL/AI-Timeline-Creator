## 三、设计方向与视觉语言

### 3.1 设计理念："织机"隐喻

Storyloom = "织机"（Loom），核心隐喻是"织布"：
- **时间轴** = 经线（纵向轨道）
- **事件** = 纬线（横向编织）
- **连线** = 编织纹理（关系连接）
- **角色/世界观** = 染料与材质

### 3.2 视觉风格：东方文艺 × 现代 SaaS 混合

结合洛笙的**文艺手账感**和 Plottr/Linear 的**现代清爽感**：

| 维度 | 方向 | 参考 |
|------|------|------|
| **色调** | 暖色为主，洛笙主题深化，其他主题保持温暖 | 洛笙创作软件 |
| **布局** | 信息密度提升，借鉴洛笙三栏思路 | 洛笙创作软件 |
| **卡片** | 微圆角 + 左侧色条 + 柔和阴影 | Plottr scene card |
| **按钮** | 大圆角，绿色/棕色双主色（洛笙主题） | 洛笙创作软件 |
| **动画** | 流畅但克制，不喧宾夺主 | Linear |
| **字体** | 系统字体优先，可选思源字体 | 洛笙/Notion |
| **质感** | 纸张纹理（极淡）、毛玻璃效果 | 洛笙/苹果 |

### 3.3 设计令牌扩展

```css
/* 新增 */
--texture-paper: 极淡的纸张纹理（opacity 0.03）
--texture-grid: 时间轴网格线（opacity 0.06）
--shadow-card-hover: 0 8px 24px -4px rgba(0,0,0,0.12)
--shadow-card-active: 0 2px 8px rgba(0,0,0,0.08)
--animation-enter: 150ms cubic-bezier(0.16, 1, 0.3, 1)
--animation-exit: 100ms ease-in
--glass-bg: rgba(var(--card), 0.7)
--glass-border: rgba(var(--border), 0.3)
```

### 3.4 色彩策略

- **时间轴事件卡片**：按轨道色 + 主题色混合，增加毛玻璃/半透明效果
- **状态指示**：废弃 `#red-500` 等硬编码，全部使用语义色（success/warning/destructive）
- **图表颜色**：StatsView / ForeshadowingGraph 使用主题色派生色阶
- **背景纹理**：洛笙主题添加极淡纸张纹理，午夜主题添加星空粒子

## 四、具体改造方案（按优先级）

### Phase 1：视觉门面（最高优先级）

#### 4.1.1 EmptyShell 重设计 ★
**现状**：文字列表 + 简单按钮，缺乏品牌感
**目标**：打造"第一眼惊艳"的欢迎界面，参考洛笙的温暖感

设计方案：
- 左侧：大幅品牌视觉（时间轴抽象图形/织机意象，SVG 绘制）
- 中央：工作区卡片网格，卡片有 Hover 动效（上浮 + 阴影加深）
- 右侧：快速操作面板（新建/导入/最近/教程入口）
- 背景：微妙的动态纹理（洛笙主题为暖色纸张，子夜为星空粒子）
- 色调：温暖米色为主，按钮绿色/棕色双主色
- 新增：首次引导（Onboarding）动画流程

参考：洛笙创作软件温暖感 + Plottr 欢迎页 + Notion 空态设计

#### 4.1.2 时间轴事件卡片重设计 ★
**现状**：矩形卡片，仅显示标题和颜色
**目标**：信息丰富、有质感、参考洛笙卡片风格

设计方案：
- 卡片形状：微圆角（12px）+ 左侧色条（轨道色，4px宽）
- 内容层：标题 + 时间 + 图标标签（角色/地点/伏笔）
- 状态层：完成度指示（底部细进度条，1-2px）
- 交互：Hover 显示快速操作（编辑/关联/大纲）
- 选中态：发光边框 + 轻微放大（scale 1.02）
- 拖拽态：半透明 + 阴影加深
- 背景：毛玻璃效果（backdrop-blur-sm）

参考：洛笙的大纲卡片 + Plottr 的 scene card + Linear 的 issue card

#### 4.1.3 TimelineCanvas 背景优化 ★
**现状**：纯色背景 + 网格线
**目标**：有"时间感"的沉浸背景，参考洛笙的纹理感

设计方案：
- 背景：极淡的纹理（洛笙主题为纸张纹理，子夜为星空微粒子）
- 时间刻度：优化标尺设计，重要日期加粗、刻度线更精致
- 当前时间线：垂直发光指示线（类似视频编辑器 playhead）
- 轨道头：毛玻璃效果（backdrop-blur）+ 底部细阴影
- 网格线：更淡、更细，不抢视觉焦点

### Phase 2：交互体验（高优先级）

#### 4.2.1 视图切换动效
**现状**：瞬间切换，无过渡
**目标**：流畅的视图切换动画

设计方案：
- 使用 Framer Motion（已有依赖）实现视图切换：淡入淡出 + 轻微缩放
- 时间轴 ↔ 大纲：共享元素动画（选中事件平滑过渡）
- 面板展开/折叠：滑入滑出 + 遮罩渐变
- 工作区切换：卡片式过渡

#### 4.2.2 骨架屏加载态
**现状**：空白或简单 spin
**目标**：有结构感的骨架屏

设计方案：
- 时间轴：灰色轨道头 + 灰色卡片占位（带圆角）
- 工作区卡片：灰色卡片网格（3 列）
- 关系图：灰色节点 + 连线占位
- 使用 shimmer 动画（从左到右的光带扫过）
- 颜色：从主题背景色派生，不突兀

#### 4.2.3 命令面板优化
**现状**：基础列表，视觉简陋
**目标**：类似 Spotlight / Raycast 的搜索体验

设计方案：
- 模糊背景（backdrop-blur-lg）+ 大阴影
- 分类分组：带图标和描述
- 选中项高亮：背景色 + 左侧色条（主色）
- 快捷键提示：右对齐，等宽字体
- 空状态：提示文字 + 快捷新建
- 顶部搜索框：大字号、无边框、带图标

参考：Raycast、VS Code Command Palette、Arc Browser

---

### Phase 3：数据可视化（中优先级）

#### 4.3.1 StatsView 图表升级
**现状**：基础 D3 图表，颜色未统一
**目标**：专业级数据可视化

设计方案：
- 继续使用 D3，但统一样式
- 颜色：从主题令牌派生色阶（主色、辅助色、强调色）
- 图表类型：热力图、柱状图、饼图、折线图统一风格
- 交互：Hover 显示 Tooltip、点击下钻
- 空状态：优雅的 "暂无数据" 插画（EmptyState 组件）
- 字体：数据标签使用等宽字体

#### 4.3.2 RelationshipGraph 美化
**现状**：D3 默认圆形节点 + 简单连线
**目标**：有辨识度的节点设计，参考洛笙的角色关系图

设计方案：
- 节点：角色 = 圆形头像（或默认头像图标）+ 名称标签
- 节点：事件 = 小卡片缩略（圆角矩形）
- 节点：世界观 = 地点图标（方形）
- 连线：粗细与关系强度成正比，颜色与主题色关联
- 选中态：高亮关联节点，其余变暗（focus+dim）
- 布局：力导向优化，减少重叠
- 背景：与时间轴一致的纹理

#### 4.3.3 树状视图升级
**现状**：基础树形图
**目标**：更清晰的决策树，参考洛笙的大纲层次

设计方案：
- 节点：卡片式，含标题和状态图标（完成/进行中/待办）
- 连线：贝塞尔曲线，条件标注在线上
- 颜色：结局类型用不同色环（好结局/坏结局/中性）
- 缩放：支持画布缩放和平移（鼠标滚轮）
- 筛选：按阶段/状态过滤节点

### Phase 4：细节打磨（低优先级）

#### 4.4.1 字体优化（可选）
- 引入 `@fontsource/noto-sans-sc` + `@fontsource/noto-serif-sc`（增加 ~200KB）
- 标题用衬线（思源宋体），正文用无衬线（思源黑体）
- 代码/数据用 JetBrains Mono
- 或保持系统字体（保持轻量，洛笙也用系统字体）

#### 4.4.2 微交互
- 按钮 Hover：轻微上移（translateY -1px）+ 阴影加深
- 开关切换：弹性动画（spring）
- 输入框聚焦：发光边框 + 轻微缩放
- Toast 通知：滑入 + 进度条
- 卡片 Hover：上浮 + 阴影
- 标签切换：下划线滑动动画

#### 4.4.3 大纲视图拆分
- 将 `OutlineView.tsx`（948 行）拆分为：
  - `OutlineTree.tsx` — 树形结构
  - `OutlineEditor.tsx` — 右侧编辑
  - `OutlineToolbar.tsx` — 顶部筛选（参考洛笙的筛选标签）
  - `useOutline.ts` — 逻辑 Hook

#### 4.4.4 伏笔面板重构
- 原生表单 → 全部 TDesign 组件
- 手写下拉 → TSelect
- 状态色 → 主题令牌
- 新增：伏笔生命周期可视化（小型时间轴缩略图）
- 参考洛笙的"呼应链条"展示方式
